Cost Breakdown

Pricing model: You pay for GPU time, not tokens. Modal charges per second of GPU usage (~$0.018/minute on A10G) regardless of token count. Each call takes 60-90 seconds: 30-60s to load the model into GPU memory, then 10-30s to generate the response.

About the model: Qwen 2.5 7B Instruct is a 7-billion parameter foundation model by Alibaba. We used LoRA (Low-Rank Adaptation) fine-tuning, which trains only ~1% of the model's parameters (10M out of 7B), making it much faster and cheaper than full fine-tuning. Training loss improved from baseline to 2.78.

Modal handles GPU provisioning, persistent model storage (Volumes), and auto-scaling inference. You pay only for actual compute time used.

Modal Infrastructure

What We Used from Modal
▼
1. Modal Volumes
Persistent distributed storage for the 14GB model weights. Download once, cache it, reuse it across every pipeline step. First load: 10 minutes. Subsequent runs: instant. Shared across all Functions.
2. Modal Functions
Serverless GPU functions for all compute. A100 (40GB) for fine-tuning, A10G for inference serving. Call them from TypeScript, they run on Modal's infrastructure. Automatic GPU provisioning and release.
3. TypeScript SDK
Orchestrated the entire pipeline from TypeScript. Python functions for GPU work, TypeScript for coordination. Single SDK for the full workflow. Type-safe function calls across languages.
4. Auto-scaling
The entire pipeline auto-scales. Need 10 parallel GPU instances? Modal spins them up. Done with training? GPU releases automatically. Scale to zero when idle. Pay only for actual compute time used.

The Pipeline

Step 01
Load Model
Downloaded Qwen 2.5 7B (~14GB) from HuggingFace to Modal Volume. Persistent distributed storage shared across all steps. First load: 10 minutes. Subsequent runs: instant (cached). Cost: $1.16.
Step 02
RL Rollouts
Ran 10 parallel inference calls to evaluate base model performance. 8/10 completed (2 timeouts). Demonstrates Modal's auto-scaling: spin up multiple GPUs in parallel, run evaluations, scale back to zero.
Step 03
Fine-tune
LoRA fine-tuning on A100 GPU. 206 Q&A pairs, 10 epochs, 9.3 minutes. Trained only ~10M parameters (1% of 7B total). Final loss: 2.78. Model saved to Volume. Cost: $0.52.
Step 04
Serve
Both models deployed on A10G GPUs with auto-scaling (zero when idle). Cold start: ~60s (loads 14GB into GPU). Warm inference: 10-30s per query. Web endpoint: dancleary54--vertical-ai-serve-inference-endpoint.modal.run
